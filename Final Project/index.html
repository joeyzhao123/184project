Final Project Proposal - Minecraft Shaders

Joey Zhao, Massimo Tseng, Jason Gong, Shawn Zhao



Project Summary

For this project, we are designing a Minecraft shaderpack. Minecraft’s existing shading is very basic and limited so we plan to add more realism and immersion to the game, mimicking real-life lighting by implementing volumetric fog, shadow mapping, screen-space reflections for more realistic water, and ray tracing.


Project Description

We are trying to solve unrealistic lighting in Minecraft and add more immersive and realistic effects to the game. Currently, shading in Minecraft simply follows a linear fall off between where a light source is and the surrounding area. By providing more realistic lighting, we can increase the quality of the player experience and add more depth to gameplay. We hope to deliver a number of shader effects including volumetric fog, shadow mapping, screen-space reflections for more realistic water, and ray tracing.
A challenge for this project is navigating pre-existing frameworks and mods used in Minecraft shading; since this project is not a standalone we will need to make sure it integrates well with the existing frameworks that we will build off of and the existing game engine. We plan to build our project off of the OptiFine mod’s rendering pipeline using OpenGL/GLSL. From there, we will implement our various shader effects based on the papers and documentation referenced in the Resources section.


Deliverables

What we plan to deliver

We plan to deliver a functioning shaderpack with implementations of volumetric fog, shadow mapping, screen-space reflections, and ray tracing.

Here are examples of what we are trying to implement. These images were taken in Minecraft using the BSL Shaderpack and a 128x realistic texture pack.

The volumetric fog algorithm uses volumetric textures as intermediate storage to decouple and parallelize the emulation of atmospheric scattering for efficient and realistic fog.
Shadow mapping creates dynamic shadows by rendering the scene from the light in the first pass and then rendering the scene with a check that it is in a shadow.
Screen space reflection lets us add realistic reflections to things like water by reflecting the screen image onto itself to create the illusion of reflection.
Finally, we will also implement realistic lighting via raytracing similar to what we have covered in class.
None of our group has experience with modding or creating shaders so we might run into trouble working with the existing frameworks and game engines. In addition, we are worried that the debugging process and development might be difficult since there are a lot of parts to integrate and make sure work together.
In terms of a demo, we could bring a laptop with our shader pack installed for viewers to test it out in Minecraft in real-time. We will also make a video showcasing the different effects that we have created.
To measure quality and performance, we can compare to other shaderpacks that are publicly available. For performance, Minecraft automatically tracks frames per second, which is a great metric to test performance. For quality, we can compare how our game looks compared to other popular shaderpacks. If we mimic similar visual effects, it will be successful.

What we hope to deliver

Ideally, we hope to have a finished product with all of the effects we have detailed working. In addition, we hope that performance is above 60 frames per second for a playable experience. If things go well and we get ahead of schedule, we ideally would like to improve the performance using fps as a metric. A good looking game has to also be playable, and increasing the framerate helps less graphically powerful devices run our shaders.
